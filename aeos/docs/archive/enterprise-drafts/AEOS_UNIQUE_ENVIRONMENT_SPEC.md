# AEOS Unique Environment Specification v1.1

## Mission

AEOS v1.1 is a unique AI-first engineering environment designed to provide governed automation across software engineering ecosystems.

It must operate as:

```text
Portable Engineering Workbench
+ Agent Runtime
+ MCP Gateway
+ Skill Factory
+ Playbook Factory
+ LCP Studio
+ Evidence OS
+ Judge Layer
+ Security Governance
+ Observability Mesh
+ Pack Marketplace
+ Evaluation Harness
```

## Product Definition

AEOS transforms repositories into controlled workspaces.

It should be able to:

1. scan an ecosystem;
2. infer stack with evidence;
3. generate environment;
4. recommend playbooks;
5. generate project-specific skills;
6. execute read-only analysis;
7. propose code/test/dependency changes;
8. apply patches only under approval;
9. test and validate;
10. rollback;
11. package evidence;
12. open controlled PRs;
13. learn lessons;
14. improve skills;
15. maintain governance.

## Unique Environment Principle

AEOS should not be a set of prompts.

AEOS should be an operational layer where every technical action is:

```text
contextualized
authorized
policy-checked
tool-routed
logged
evidenced
judged
packaged
auditable
reversible when mutable
```

## Non-Negotiables

- deterministic gates before LLM judgement;
- deny-all by default;
- no ungoverned tool access;
- no raw secret persistence;
- no unsupported claims;
- no finalization without evidence;
- no mutation without rollback;
- no agent self-judging;
- no direct import of untrusted packs;
- no production mutation without explicit human approval.
