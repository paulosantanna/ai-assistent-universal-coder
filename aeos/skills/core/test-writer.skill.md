# Test Writer Skill

**ID:** test-writer
**Version:** 1.0.0
**Owner Agent:** tester
**Risk Level:** low

## Mission
Generate proposed test files in sandbox directory based on evidence from source code analysis. Never writes to real test directories.

## Scope
- Generate test files in .aeos/sandbox/{execution_id}/generated-tests/
- Follow project's test framework and patterns
- Cite source evidence in each test file

## Allowed Actions
- `filesystem.read` — read source files as evidence
- `filesystem.write_aeos` — write to .aeos/sandbox/{execution_id}/
- `generate_evidence` — register generated tests

## Forbidden Actions
- Write to src/test/, tests/, or any real test directory
- Modify existing test files
- Run test commands
- Commit test files

## Required Capabilities
- READ_REPOSITORY
- WRITE_SANDBOX
- GENERATE_TEST
- GENERATE_REPORT

## Required Evidence
- Source files used as evidence (SHA-256)
- Generated test files (SHA-256)
- Test index file

## Prompt Contract

- State the objective, target scope, assumptions and constraints before execution.
- Use only evidence-backed facts; mark uncertainty explicitly.
- Route tool access through approved command, MCP or Tool Router paths.
- Redact secrets, credentials, tokens and sensitive values.
- Return facts, assumptions, risks, recommendations, evidence_refs and blocking_conditions when applicable.
- Stop when required evidence, permissions, policy approval or input context is missing.

## Quality Gates
- Every generated test file MUST have a header with source citation
- Test framework must match project's detected framework
- Tests must be syntactically valid for the target framework

## Output Schema
Each test file must include:
```
# Source: <relative path to real source file>
# Evidence: SHA-256 of source file
# Generated by: test-writer skill v1.0.0
```

## Blocking Conditions
- Test file does not cite its source
- Test file written outside .aeos/sandbox/
- Test framework mismatch without documentation

## Examples of Allowed Behavior
- Generate test_user_service.py in sandbox citing src/services/user_service.py
- Generate test_auth.py based on src/auth.py with test cases for all functions

## Examples of Forbidden Behavior
- Writing test_user_service.py to src/tests/
- Modifying an existing real test file
- Running pytest on generated tests
