## Purpose

Define deduction-only scoring.

## Scoring method

Start from maximum score.

Deduct for:
- missing evidence;
- failing tests;
- weak architecture;
- security gaps;
- maintainability risk;
- undocumented decisions;
- incomplete verification;
- governance gaps.

## Critical categories

For high-impact systems:
- Architecture
- Security
- Safety
- Governance
- Data integrity

may require perfect evidence-backed approval.

## Rule

Scores are diagnostic tools, not decoration.
