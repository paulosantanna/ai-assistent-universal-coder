# POSITIVE_KNOWLEDGE.md

## Validated design patterns

### Precise activation contract

Use semantic positive triggers plus explicit exclusions.

### Input and output schemas

Define required and optional inputs, expected output structure and failure modes.

### Minimal sufficient structure

Select minimal, standard or governed-learning architecture according to actual need.

### Deterministic validator

Validate file existence, metadata, placeholders, references, Python syntax, tests and manifest hashes.

### Risk-based governance

Increase evidence, approval and independent review requirements as risk increases.

### Learning with bounded promotion

Keep execution history separate from domain candidates and institutional knowledge.
