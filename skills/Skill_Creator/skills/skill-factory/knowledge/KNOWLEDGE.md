# KNOWLEDGE.md

## Purpose

Canonical design knowledge for skill creation.

## Principles

- Reuse begins with a stable recurring objective.
- Activation precision is as important as execution quality.
- Minimal sufficient architecture is preferred.
- Deterministic checks belong in code.
- Evidence and authority boundaries must be explicit.
- Memory requires a reader, writer, schema and promotion process.
- A generated package must be installable, not merely descriptive.
